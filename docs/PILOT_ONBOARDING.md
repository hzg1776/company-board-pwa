# Pilot Customer Onboarding

## Goal

Get one pilot customer live with minimal operator error and a repeatable handoff.

## Ideal Pilot Customer

- one location or one team
- low internal Systems friction
- clear need for employee notices or status updates
- one accountable admin contact

## Pilot Scope

Included:

- HR admin setup
- Systems setup
- employee login setup
- employee feed
- weather updates
- publishing notices
- push notification enrollment

Not included:

- multi-tenant separation
- enterprise SSO
- advanced analytics
- custom reporting

## Pre-Onboarding Checklist

1. confirm hostname and public URL
2. confirm reverse proxy routing
3. confirm backup script works
4. confirm health checks pass
5. confirm HR and Systems credentials are set
6. confirm one named customer admin

## Customer Setup Checklist

1. create HR admin access
2. provision Systems separately
3. create initial employee accounts
4. publish first notice
5. verify employee view
6. test push enrollment on one real device

## Day 1 Success Criteria

- admins can sign in
- employees can sign in
- one notice is published successfully
- one device receives push successfully
- operator can back up data and pass health checks

## Support Boundaries

Pilot support is best handled with:

- one support email or phone path
- one operator owner
- one documented release window

## Expansion Triggers

Consider paid expansion when the pilot customer asks for:

- multiple departments or sites
- role separation beyond current pilot scope
- analytics or reporting
- audit exports
- automated onboarding
